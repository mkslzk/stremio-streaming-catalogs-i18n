# Internationalization (i18n) Configuration

This fork replaces hardcoded English metadata from Cinemeta with TMDB in 80+
languages. The language is set server-side via the `CATALOG_LANGUAGE` env var.

## Quick reference

```bash
# German metadata
CATALOG_LANGUAGE=de node --env-file=.env index.js

# French metadata
CATALOG_LANGUAGE=fr node --env-file=.env index.js

# Japanese metadata
CATALOG_LANGUAGE=ja node --env-file=.env index.js
```

Default: `en` (English).

## How it works

```
┌──────────────────┐    ┌────────────────┐    ┌──────────────────┐
│ JustWatch        │ ─► │ metaEnricher   │ ─► │ Stremio client   │
│ (catalog list)   │    │ (TMDB → Cine.) │    │ (your language)  │
└──────────────────┘    └────────────────┘    └──────────────────┘
        IMDB-IDs            titles in your language
```

1. **JustWatch** provides the catalog (which titles are available on Netflix,
   Disney+, etc., in your country)
2. **`metaEnricher`** (this fork) looks up each title in TMDB using the IMDB-ID
   and replaces the English-only `name`/`description` with the localized
   version in your language
3. If TMDB is down or rate-limited, **Cinemeta** is used as fallback

## TMDB credentials

You need a TMDB API key. Get one (free) at https://www.themoviedb.org/settings/api

Either:

- **`TMDB_API_KEY`** — v3 API Key (recommended for server-side use, stable, no
  refresh)
- **`TMDB_READ_TOKEN`** — v4 Read Access Token (alternative; needs
  `api_read` scope)

Place in `.env`:

```
TMDB_API_KEY=your_v3_api_key_here
# or: TMDB_READ_TOKEN=your_v4_read_token_here
```

## Caching

Two cache files speed up repeated boots:

- **`cache/catalog-cache.json`** — the JustWatch catalog list (12 MB,
  refreshes every 6h by default)
- **`cache/tmdb-id-cache.json`** — IMDB → TMDB-ID mappings (~50 KB per 2000
  titles, **persistent across boots**)

The TMDB cache is the key to avoiding TMDB rate limits. The first boot maps
all IMDB-IDs in your catalogs to TMDB-IDs (~7000 calls, takes ~1 minute,
may hit rate limits on free tier). Every subsequent boot loads the cache from
disk and only fetches metadata for new titles.

## Frontend language selection (Phase 4)

The Vue frontend at `vue/src/App.vue` has a language dropdown with the 21 most
popular languages. The selected language is **stored in the Stremio config
string** (as the 8th colon-separated field) and forwarded to the manifest
endpoint.

> **Note:** for v1, the **server still uses** `CATALOG_LANGUAGE` for actual
> metadata enrichment — the per-user-config language is reserved for a future
> multi-tenant deployment.

## Troubleshooting

### All titles are still in English

Check your `.env` is loaded: `node --env-file=.env -e "console.log(process.env.CATALOG_LANGUAGE)"` should print your language. If it prints `undefined`, the file is missing or has a typo.

### HTTP 429 errors during cold boot

TMDB free tier allows ~40 req/sec. The cold-boot burst can throttle you. Wait
5–10 minutes and re-launch — the cache will be saved on graceful shutdown.

### TMDB 401 errors

Your API key may have been deactivated, or the v4 token doesn't have
`api_read` scope. Generate a fresh v3 key at https://www.themoviedb.org/settings/api
